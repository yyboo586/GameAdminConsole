# 前端路由配置示例（仅供开发测试）

> **重要提示**：这个配置仅用于开发环境快速测试，生产环境必须使用后端控制路由！

## 快速测试方法

如果你想在开发环境中快速测试游戏审核功能，而不想等待后端配置菜单，可以临时在前端路由文件中添加测试路由。

### 方法 1: 在 route.ts 中添加临时路由

编辑 `src/router/route.ts` 文件，在 `dynamicRoutes` 的第一个路由的 `children` 数组中添加：

```typescript
// 在 src/router/route.ts 的 dynamicRoutes[0].children 中添加

{
    path: '/system/tGame/review',
    name: 'systemTGameReview',
    component: () => import('/@/views/system/tGame/review/index.vue'),
    meta: {
        title: '游戏审核',
        isLink: '',
        isHide: false,
        isKeepAlive: true,
        isAffix: false,
        isIframe: false,
        roles: ['admin'],
        icon: 'ele-Check',
    },
},
```

			{

### 方法 2: 使用浏览器直接访问

如果已经添加了路由，可以直接在浏览器地址栏访问：

```
http://localhost:你的端口号/#/system/tGame/review
```

例如：
```
http://localhost:5173/#/system/tGame/review
```

## 测试步骤

1. **添加路由**（使用上述方法 1）
2. **重启开发服务器**
   ```bash
   npm run dev
   ```

3. **访问页面**
   - 在浏览器中访问：`http://localhost:端口/#/system/tGame/review`
   - 或者在左侧菜单中应该能看到"游戏审核"菜单项

4. **测试功能**
   - 查看审核列表
   - 点击"查看详情"查看游戏信息
   - 测试"通过审核"和"拒绝审核"功能

## 注意事项

⚠️ **生产环境配置**

这个配置**只用于开发测试**，生产环境必须：
1. 删除或注释掉这段临时路由代码
2. 在后端系统的菜单管理中正确配置菜单
3. 让后端接口返回菜单数据

⚠️ **权限控制**

前端路由配置的权限控制比较弱，真正的权限控制应该：
1. 在后端验证用户权限
2. 通过后端菜单接口返回用户有权访问的菜单
3. 前端根据后端返回的菜单动态生成路由

⚠️ **菜单显示**

如果使用前端路由配置，菜单可能不会自动出现在左侧导航栏中，这是正常的。可以：
1. 直接通过 URL 访问
2. 或者配置后端菜单让菜单正常显示

## 恢复到后端控制路由

测试完成后，记得：

1. **删除或注释** `src/router/route.ts` 中添加的临时路由代码

2. **在后端配置菜单**（参考《游戏审核功能说明.md》）

3. **重启前端服务**
   ```bash
   npm run dev
   ```

4. **清除浏览器缓存**（如果菜单显示异常）

## 常见问题

### Q1: 添加路由后页面空白？
**A**: 检查控制台是否有错误，确认组件路径是否正确。

### Q2: 菜单没有显示？
**A**: 前端临时路由不会自动生成菜单，可以直接通过 URL 访问，或者配置后端菜单。

### Q3: 提示权限不足？
**A**: 检查路由配置中的 `roles` 是否包含你当前登录的角色。

### Q4: 接口请求失败？
**A**: 检查 `src/api/system/tGame.ts` 中的 `GameEngine` 地址是否正确。

## 推荐的开发流程

**最佳实践：**

1. **第一阶段**（快速开发测试）
   - 使用前端临时路由快速测试功能
   - 验证页面展示和交互逻辑
   - 调试接口调用

2. **第二阶段**（正式配置）
   - 删除前端临时路由
   - 在后端配置正式菜单
   - 测试后端路由控制是否正常

3. **第三阶段**（完善权限）
   - 配置细粒度权限
   - 测试不同角色的访问权限
   - 验证按钮级别的权限控制

---

**最后提醒**：生产环境务必使用后端控制路由，前端临时路由仅用于开发测试！
